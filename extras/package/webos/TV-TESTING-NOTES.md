# VLC on LG webOS TV — Testing Notes

Findings from live testing on a real LG TV running webOS 4.x.
Covers device environment, each startup failure, root cause, and fix applied.

---

## Device Environment

| Property         | Value |
|------------------|-------|
| Device           | LG TV, webOS 4.x |
| TV IP / SSH port | `192.168.0.109:9922` |
| SSH user         | `prisoner` (no password — dev mode) |
| SSH key          | `~/.ssh/lgtv_plain` |
| App install path | `/media/developer/apps/usr/palm/applications/org.videolan.vlc.webos/` |
| CPU arch         | ARMv7-A (32-bit, `armhf`) |
| Wayland socket   | `/tmp/xdg/wayland-0` |
| Shell protocol   | `wl_shell` (NOT `xdg_wm_base` / `xdg_shell`) |

Kernel / libc observed at runtime:
- Linux 5.4.96 (typical for webOS 7; 4.x TVs have similar ABI)
- glibc 2.31
- `libstdc++.so.6` present but old (lacks `GLIBCXX_3.4.32`, which GCC 14 needs)

## Problem Log

### Problem 1 — `libstdc++.so.6: GLIBCXX_3.4.32 not found`

**Symptom:** VLC process exits immediately. Journal shows:
```
vlc: /usr/lib/libstdc++.so.6: version `GLIBCXX_3.4.32' not found
```

**Root cause:** The buildroot-nc4 toolchain uses GCC 14, which produces binaries
that require `GLIBCXX_3.4.32`. The TV ships an older `libstdc++.so.6`.

**Fix:** Bundle the toolchain's own `libstdc++.so.6` and `libgcc_s.so.1` inside
the IPK under `vlc/lib/`. `package.sh` now copies them from
`$WEBOS_TOOLCHAIN/arm-webos-linux-gnueabi/lib/` when `WEBOS_TOOLCHAIN` is set.

---

### Problem 2 — Qt plugin fails to load (wrong shell integration)

**Symptom:** Log contains:
```
interface "qt" initialization failed
qt: cannot load libqt_plugin.so
```
VLC exits immediately after; no window appears.

**Root cause (a) — Wayland shell mismatch:**
Qt6 defaults to `xdg-shell` (`xdg_wm_base`). webOS 4.x compositor only advertises
`wl_shell`. Without explicit shell selection Qt prints:
```
qt.waylandclient.shellintegration: Failed to find a shell integration
```
and refuses to create a surface.

**Fix:** Set in `run.sh`:
```sh
export QT_WAYLAND_SHELL_INTEGRATION=wl-shell
```

**Root cause (b) — QML static module archives dropped by linker:**
The `libqml_module_*.a` convenience archives (compiled QML cache + qmldir) were
built into the Qt plugin as `LIBADD` static libraries. The linker silently discarded
all their symbols because they appeared to have no live references from `.o` files
(Qt's constructor-based registration idiom fools the linker's dead-code pass).

**Symptom:** `libqt_plugin.so` loaded successfully, `wl-shell` connected, but no
QML module constructors ran. VLC crashed inside `QQmlEngine`.

**Fix:** Force `--whole-archive` for those archives via libtool `.la` stubs with
`shouldnotlink=yes`. `extras/buildsystem/patch-qt-makefile.py` creates
`libqml_module_*.la` stubs and patches `libqt_plugin_la_LIBADD` in the build-dir
`Makefile`. Verified via `make V=1` that `--whole-archive … --no-whole-archive`
appears in the final `g++` invocation wrapping all 11 `libqml_module_*.a` files.

**Scratch-build survival:** Both `patch-qt-makefile.py` and `embed-qml-sources.py`
(see Problem 3) operate on the generated build directory, which is wiped on a clean
rebuild. To make the fix automatic, `build-webos.sh` calls both scripts as a
post-build step immediately after `make` completes, then re-runs just the plugin
link. A scratch build therefore goes through:
```
extras/package/webos/build-webos.sh all
  └── make (compiles everything, links libqt_plugin.so without the fix)
  └── patch-qt-makefile.py --build-dir <build>/modules/gui/qt ...
  └── embed-qml-sources.py --build-dir <build>/modules/gui/qt ...
  └── rm .libs/libqt_plugin.so libqt_plugin.la
  └── make libqt_plugin.la  (re-links with --whole-archive + QML resources)
```

### Problem 3 — `No such file or directory: qrc:/qt/qml/VLC/MainInterface/MainInterface.qml`

**Symptom (first working `libqt_plugin.so` load):**
```
qml loading No such file or directory qrc:/qt/qml/VLC/MainInterface/MainInterface.qml:4294967295
no suitable compositor found
unable to create main interface
```
Qt loaded, Wayland surface created (`wl-shell` confirmed), but VLC exited without
showing a window.

**Root cause:** Two separate things in VLC's Qt build chain were confused:

1. `*_qmlassets.cpp` (generated by `qmlcachegenr`) registers **only** the binary
   `qmldir` descriptor at `:/qt/qml/VLC/<Module>/qmldir`. The `.qml` source files
   are **not** embedded there.

2. `*_qmlcache_loader.cpp` registers pre-compiled bytecode via a cache hook
   (`QQmlAbstractUrlInterceptor` subclass). The hook is only called *after* Qt
   has resolved the source URL. Because Qt's resource system returns "not found"
   for `qrc:/qt/qml/VLC/MainInterface/MainInterface.qml`, the hook is never
   reached.

3. The `qmldir` files contain `prefer :/qt/qml/VLC/MainInterface/`, which
   suppresses any filesystem fallback. Qt only looks in the resource system
   and fails.

**Fix:** Generate a combined `.qrc` embedding all 241 `.qml` source files under
their correct resource paths (`/qt/qml/VLC/<Module>/<File>.qml`), compile it to
an ARM object via the SDK's `rcc` + cross `g++`, archive it as
`libvlc_qml_sources.a`, add a `.la` stub, and add it to `libqt_plugin_la_LIBADD`.
Because it is treated as a convenience library (`shouldnotlink=yes`) it also gets
wrapped in `--whole-archive`, ensuring the Qt resource initializer constructor
actually runs at load time.

Script: `extras/buildsystem/embed-qml-sources.py`

### Problem 4 — `libgmp.so.10 not found`, `libvlc_pulse.so.0 not found`

**Symptom:** Plugin-cache log at startup:
```
plugin_Load: cannot load libgnutls_plugin.so (libgmp.so.10 not found)
plugin_Load: cannot load libpulse_plugin.so (libvlc_pulse.so.0 not found)
```

**Status:** Non-fatal. GnuTLS and PulseAudio plugins simply don't load.
`libgmp.so.10` is a GnuTLS dependency from contribs. The TV has no PulseAudio.

**Fix for libgmp:** `package.sh` now bundles all `*.so.*` files from
`$WEBOS_DEPS_PREFIX/lib/` (excluding system libc/libm/libpthread/etc.) into
`vlc/lib/` automatically.

**Fix for PulseAudio:** Not fixable; webOS has no PulseAudio server. The plugin
is harmlessly skipped.

## Environment Variables Required in `run.sh`

The following must be set in the app's launch script. `package.sh` generates
`run.sh` with all of these included:

```sh
# Wayland — use wl_shell (webOS 4.x does not expose xdg_wm_base)
export QT_WAYLAND_SHELL_INTEGRATION=wl-shell
export QT_QPA_PLATFORM=wayland

# Wayland socket path (webOS puts it here, not /run/user/...)
export XDG_RUNTIME_DIR=/tmp/xdg
export WAYLAND_DISPLAY=wayland-0

# Qt paths (all under the app bundle)
export QT_PLUGIN_PATH="${APP_DIR}/qt6/plugins"
export QT_QPA_PLATFORM_PLUGIN_PATH="${APP_DIR}/qt6/plugins/platforms"
export QML2_IMPORT_PATH="${APP_DIR}/qt6/qml"
export QT_QUICK_CONTROLS_STYLE=Basic

# Locale — webOS locale settings can confuse Qt's number formatting
export QT_NO_SETLOCALE=1
export LANG=C.UTF-8
export LC_ALL=C.UTF-8

# VLC paths
export VLC_PLUGIN_PATH="${APP_DIR}/vlc/plugins"
export VLC_LIBEXEC_PATH="${APP_DIR}/libexec/vlc"
export VLC_DATA_PATH="${APP_DIR}/share/vlc"

# Skip Qt platform check (vlc-qt-check tries X11 on webOS, which always fails)
export VLC_QT_SKIP_CHECK=1
```

## Build Artefacts Required for Full Qt GUI

`build-webos.sh build` (or `extras/package/webos/build-webos.sh all`) applies both fixups automatically
after `make` completes.  No manual steps are needed on a scratch build.

If you need to re-apply them manually to an existing build directory (e.g. after
editing a single `.qml` file):

```bash
# 1. Create libtool stubs so --whole-archive wraps the QML module archives
python3 extras/buildsystem/patch-qt-makefile.py \
    --build-dir ~/vlc-webos-build/modules/gui/qt

# 2. Embed .qml source files as Qt resources (required for Qt to locate them)
python3 extras/buildsystem/embed-qml-sources.py \
    --build-dir ~/vlc-webos-build/modules/gui/qt \
    --src-dir   /path/to/vlc/source \
    --toolchain ~/vlc-webos-build/sdk/arm-webos-linux-gnueabi_sdk-buildroot

# 3. Relink
cd ~/vlc-webos-build/modules/gui/qt
rm -f .libs/libqt_plugin.so libqt_plugin.la
PATH=/usr/bin:/path/to/sdk/bin make libqt_plugin.la

# 4. Copy updated plugin into deploy tree
cp .libs/libqt_plugin.so ~/vlc-webos-deploy/lib/vlc/plugins/gui/
```

## Deployment Quick Reference

```bash
# Push only the Qt plugin (fast, no repackage needed)
ares-push -d lgtv \
  ~/webos-package/stage/vlc/plugins/gui/libqt_plugin.so \
  /media/developer/apps/usr/palm/applications/org.videolan.vlc.webos/vlc/plugins/gui/

# Or do a full repackage + install

ares-launch -d lgtv org.videolan.vlc.webos
```
